Notes for testers and developers

  • The table demo defines a row-click event. This is executed when the user clicks on the row. However, this functionality shows an alert that the 'Id' and 'Status' are being posted. But the backend is not wired to post data.
  • The table demo defines a row-double click event. This is executed when the user double-clicks on the row. However, this functionality shows an alert that the 'Id' and 'Status' are being posted. But the backend is not wired to post data.
  • Horizontal scroll will be applied for a table width exceeding 1100px and vertical scroll will trigger for a height that exceeds 750px.
  • Pagination view is also set to 1100px and is not visible through out the scroll-width.
  • Replacing data with exact keys will not break the application.
